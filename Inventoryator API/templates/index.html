Inventoryator Rest Client API

/inventory/
GET
Returns a list of all inventories available as list of objects.
POST
Creates a new inventory.
Post body must be of form {"name": str, "description": str}
Data returned includes assigned database id.
/inventory/(int:id)/
GET
Returns inventory with given as object.
PUT
Updates the inventory with the given id.
Put body must be of form {"name": str, "description": str}, though only desired updates need to be included.
Data returned is the updated inventory data.
DELETE
Deletes the inventory with the given id from the database.
/inventory/(int:id)/items/
GET
Returns a list of items in the inventory with the given id, as a list of objects.
POST
Adds the provided item to the inventory with the given id.
Post body must be of the form {"name": str, "number": int}
Returns dadta for the item added, including its database id.
/inventory/(int:id)/share/
GET
Returns a share code.
/items/(int:id)/
GET
Gets the item
PUT
Edits the item.
Post body must be of the form {"name": str, "number": int}
DELETE
Deletes the item with the given id from the database.
/user/
GET
Returns a list of all users.
PUT
Login the user.
Post body must be of the form {"username": str, "password": sha256}
POST
Create a user.
Post body must be of the form {"username": unique str, "password1": sha256, "password2": sha256}
/user/(str:username)/
GET
Returns the user.
PUT
Update the user's password.
Post body must be of the form {"password": sha256, "password1": sha256, "password2": sha256}
/user/(str:username)/inventory/
GET
Returns the user's inventories.
/user/(str:username)/inventory/(int:inventory_id)/
PUT
Adds an inventory to the user by id.
DELETE
Removes an inventory from a user's account. If no users have access to the inventory, it will be deleted.
/user/(str:username)/share/(str:share_code)/
PUT
Adds an inventory to a user's account using a share code.
